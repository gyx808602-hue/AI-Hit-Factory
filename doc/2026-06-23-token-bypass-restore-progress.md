# 2026-06-23 前端 token 放开开关恢复进展

## 已完成

- 已将开发环境中的前端 token 放开开关关闭：
  - `.env.development`
  - 当前值：`VITE_BYPASS_TOKEN_CHECK='false'`
- 本次恢复仅修改环境配置，没有删除此前的可逆代码开关逻辑。
- 这意味着当前开发环境已经恢复为默认严格模式：
  - 无 `accessToken` 时，前端受保护页面会重新按未登录处理
  - 页面准入判断恢复正常

## 当前判断

- 这次恢复采用的是“关开关”而不是“删实现”，好处是后续如果你还要临时放开，只需要把环境变量改回 `true` 并重启前端服务。
- 从维护角度看，这种做法比反复改业务代码更稳，也更不容易引入回归。

## 恢复方式确认

1. 当前状态：`VITE_BYPASS_TOKEN_CHECK='false'`
2. 如需再次放开：
   - 将 `.env.development` 改回 `VITE_BYPASS_TOKEN_CHECK='true'`
   - 重启前端开发服务

## 下一步建议

1. 你现在如果继续访问受保护页面，预期会重新走登录态判断，这是正常恢复结果。
2. 如果后面你想恢复的不只是“页面准入”，还包括“401 不跳登录”之类的联调行为，需要单独做另一层可逆开关。
