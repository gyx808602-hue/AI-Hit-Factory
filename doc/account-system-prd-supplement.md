# 账号体系 PRD 与项目约束补充

更新时间：2026-06-22

来源文档：`F:\AAA_AI_aisperce\产品说明-aispeace\1. 电商AI内容生产平台_账号体系功能设计文档_V1.0.docx`

## 1. 项目定位

AI爆款工厂是面向电商内容生产的 SaaS 平台。账号体系不是单纯注册登录模块，而是身份识别、企业协作、积分计费、功能权限、内容合规和风控审计的基础底座。

MVP 第一阶段最小闭环：

用户注册 -> 绑定手机号 -> 选择个人使用或企业认证 -> 按身份开通功能 -> 使用积分生成内容 -> 系统记录任务、协议和日志。

## 2. 技术约束

- 前端技术方向：React + TypeScript。
- 页面实现顺序：先固化 OpenSpec 与 PRD 约束，再使用 Figma MCP 扫描设计稿，还原页面与组件。
- Figma 定位：Figma 是视觉和交互输入，不替代业务规则。
- API 策略：真实后端未完成前，使用 Mock API 和集中 TypeScript 类型保证页面闭环。
- 权限原则：前端负责隐藏入口和展示引导，后端必须再次校验权限。
- MVP 权限模型：不做复杂 RBAC，先支持企业管理员、创作人员、观察人员三类角色。
- 安全原则：身份证、证照、手机号等敏感信息必须脱敏展示；协议签署必须记录版本和快照。

## 3. 用户与身份模型

平台至少包含五类用户：

| 用户类型 | 说明 | 核心权限 |
| :--- | :--- | :--- |
| 个人用户 | 手机号或微信授权注册的普通用户 | 商品卖点提取、脚本生成、图文生成视频、普通模板数字人视频 |
| 实名个人用户 | 完成实名认证并签署相关协议 | 可使用本人肖像数字人、真人素材改编等部分高风险能力 |
| 企业用户 | 企业认证通过后的企业主账号 | 企业空间、子账号、企业素材库、企业积分、爆款视频改编 |
| 企业子账号 | 归属于企业空间的成员 | 按角色使用企业资源、积分和任务 |
| 平台管理员 | 平台运营与审核人员 | 企业审核、实名审核、协议管理、冻结账号、日志查询 |

设计原则：不要只在 `user` 表中使用 `user_type` 区分企业用户，企业资产、成员、积分和任务应挂到 `organization/enterprise_space` 组织空间下。

## 4. 首批 React 页面清单

| 页面 | 端 | 功能要求 |
| :--- | :--- | :--- |
| 注册/登录页 | PC/小程序 | 手机号密码、手机号验证码、微信扫码/授权登录 |
| 绑定手机号页 | PC/小程序 | 微信登录后强制绑定手机号 |
| 个人中心 | PC/小程序 | 账号信息、手机号、微信绑定、认证状态、协议记录 |
| 实名认证页 | PC/小程序 | 提交姓名、身份证信息、协议签署 |
| 企业认证申请页 | PC 优先 | 企业资料、证照材料、法人信息、联系人手机号验证 |
| 企业空间首页 | PC | 企业信息、积分余额、成员数量、近期任务 |
| 成员管理页 | PC | 创建子账号、设置角色、停用成员 |
| 积分用量页 | PC | 企业积分余额、消耗记录、成员用量 |
| 协议弹窗/签署页 | PC/小程序 | 首次使用高风险功能时强制签署 |
| 权限拦截弹窗 | PC/小程序 | 未实名、未签署、未认证、无角色权限时引导完成前置流程 |

## 5. 状态与权限约束

账号状态：

- 正常
- 未绑定手机号
- 待认证
- 认证失败
- 风控限制
- 冻结
- 注销

实名认证状态：

- 未认证
- 认证中
- 已认证
- 认证失败
- 已冻结

企业认证状态：

- 未申请
- 待审核
- 审核通过
- 审核驳回
- 已冻结
- 已注销

高风险功能使用要求：

| 功能 | 个人未实名 | 个人已实名 | 企业已认证 |
| :--- | :--- | :--- | :--- |
| 商品视频生成 | 可用 | 可用 | 可用 |
| 图文生成视频 | 可用 | 可用 | 可用 |
| 数字人模板视频 | 可用 | 可用 | 可用 |
| 爆款结构分析 | 可用 | 可用 | 可用 |
| 真人换人物 | 不可用 | 可用 + 协议 | 可用 + 协议 |
| 上传真人数字人 | 不可用 | 可用 + 协议 | 可用 + 协议 |
| 企业子账号 | 不可用 | 不可用 | 可用 |

典型权限拦截文案：

> 该功能涉及真人肖像及视频素材改编。为保障内容合规，请先完成实名认证并签署《肖像权及素材授权承诺书》。

按钮：去实名认证 / 暂不使用。

## 6. API 契约建议

首阶段前端可先使用 Mock API，但接口语义需要提前稳定：

| 模块 | 建议接口 | 说明 |
| :--- | :--- | :--- |
| 登录注册 | `POST /auth/login/password` | 手机号密码登录 |
| 登录注册 | `POST /auth/login/sms` | 手机号验证码登录 |
| 微信绑定 | `POST /auth/wechat/bind-phone` | 微信身份绑定手机号 |
| 账号资料 | `GET /account/profile` | 获取身份、状态、绑定、协议记录 |
| 实名认证 | `POST /auth/real-name` | 提交实名资料或三方认证结果 |
| 企业认证 | `POST /enterprise/applications` | 提交企业认证申请 |
| 企业空间 | `GET /enterprise/workspace` | 获取企业空间首页摘要 |
| 成员管理 | `POST /enterprise/members` | 创建企业子账号 |
| 积分用量 | `GET /points/usage` | 查询个人或企业积分用量 |
| 协议签署 | `POST /agreements/sign` | 记录协议版本、快照、IP、设备 |

## 7. 验收标准

- 注册登录：手机号验证码、手机号密码、微信登录、小程序授权均可完成登录；微信登录必须绑定手机号。
- 企业认证：用户可提交完整资料；后台可审核通过/驳回；驳回原因可回显。
- 企业空间：审核通过后自动创建企业空间，企业管理员可进入企业工作台。
- 子账号：企业管理员可创建子账号、设置角色、停用成员。
- 权限控制：个人未实名用户无法进入真人爆款改编等高风险功能。
- 积分记录：每次生成任务能记录消耗用户、企业、任务、积分和结果。
- 协议签署：高风险功能使用前必须签署对应协议，并记录版本、时间、IP。
- 日志审计：关键操作均能在后台查询到操作人、时间、对象和结果。
