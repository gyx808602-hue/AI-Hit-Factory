# 数字人视频新建弹窗预览优化实施计划

> 目标：优化“新建数字人视频”弹窗的布局与预览体验，让弹窗高度受控、内部可滚动、画布预览位于视觉中心，并让数字人、背景色、背景图联动展示。

## 一、改动范围

### Frontend
| 模块层级 | 文件路径 | 修改逻辑简述 |
| :--- | :--- | :--- |
| Frontend | `src/pages/DigitalHumanVideoTasksPage.tsx` | 重排新建弹窗内部结构，限制弹窗内容最大高度为视口约 60%，改为内部滚动；把画布预览升级为中轴区域，并联动数字人素材、背景色、背景图展示。 |
| Frontend | `src/features/digital-human/video/canvas.ts` | 如有必要，补充轻量预览辅助逻辑，保持拖拽缩放与预览尺寸换算一致。 |
| Frontend | `src/pages/DigitalHumanVideoTasksPage.test.tsx` | 新增或调整测试，覆盖弹窗高度/滚动结构、背景联动展示、数字人预览联动展示与现有拖拽缩放能力不回退。 |

## 二、实施方案

### 方案 A：保留三列，仅放大中间列
- 做法：
  - 继续使用三列布局。
  - 只增加弹窗 `bodyStyle` / `styles` 和中间预览列宽度。
- 优势：
  - 代码变动较小。
- 风险：
  - 预览依旧被左右字段区割裂，视觉中心不稳定。
  - 交互改善有限，无法充分体现“所见即所得”。

### 方案 B：保留弹窗，改成“左右配置 + 中轴预览”的稳定结构
- 做法：
  - 给 Modal 内容区设置 `max-height: 60vh` 附近约束。
  - 内容区改为内部滚动容器。
  - 预览区改成居中的固定视觉舞台。
  - 左侧放基础信息/TTS，右侧放画布/背景/高级配置。
  - 预览舞台按优先级展示：背景色 -> 背景图 -> 数字人预览素材 -> 可拖拽人物框。
- 优势：
  - 体验收益最大。
  - 不改接口契约。
  - 可复用现有 `canvas.ts`。
- 结论：
  - 采用方案 B。
