# 2026-06-27 追爆任务详情页优化方案

## 本轮诉求整理

1. 每一步要有表单联动，生成提示词前必须先保存表单。
2. 参考视频预览要使用标准比例。
3. 商品图、人物图上传后的展示要优化，输入框输入与图片回显联动，删除按钮用组件库方式优化。
4. 视频生成后再次进入详情页的展示态先可暂缓，但需要先优化列表页“查看成品”。
5. 详情页底部“下一步 / 保存”要悬浮，不要被页面内容挤到底部。

## 现状判断

- 当前 `VideoRemixTaskDetailPage.tsx` 已有三步流程，但交互仍偏“单页表单堆叠”。
- `generate-prompt` 已经会先调用保存，但“下一步切换”和“步骤间草稿同步”还不够明确。
- 参考视频预览当前是固定高度，不是标准视频容器。
- 商品图、人物图目前仍是：
  - URL 文本字段承载数据
  - 下方用自定义卡片做预览
  - 删除按钮样式较弱
- 列表页结果列当前只是普通链接，缺少“成品卡片入口感”。
- 底部操作区仍在表单流内，滚动时会被内容推到页面底部，不利于长表单操作。

## 方案对比

### 方案 A：在现有详情页上做增强式优化

- 保留现有三步结构与 API 链路。
- 在详情页内补强步骤切换保存、素材展示、底部悬浮操作区。
- 列表页只优化“查看成品”入口样式与行为。

优点：
- 改动集中，风险最低。
- 不需要改后端接口语义。
- 可以较快验证你提的 5 个点。

缺点：
- “视频已生成后的纯展示态”这一块先只做铺垫，不会一次做成完整双态页面。

### 方案 B：把详情页拆成“编辑态 / 成品态”双模式

- 已生成视频时默认进入展示态。
- 未生成时进入编辑态。
- 列表页成品查看与详情页展示态共用同一套展示结构。

优点：
- 产品感更完整。
- 后续“查看成品”与“继续编辑”边界更清晰。

缺点：
- 本轮范围会明显扩大。
- 需要一起处理更多状态边界，当前不适合和前 4 个基础交互优化混在一轮做。

## 推荐方案

推荐先走 **方案 A**。

原因：
- 你这轮提的重点，本质上是“表单联动 + 预览体验 + 页面操作区布局 + 列表成品入口优化”。
- 这些都可以在现有页面结构上稳定落地。
- “视频生成后进入展示态”目前你也明确说了可以先不处理，所以先把基础体验打磨好更合适。

## 拟定改动指向

### 1. 表单联动与步骤切换

目标：
- 点击“下一步”前先校验并保存当前表单。
- 点击“生成 Prompt”前继续保持“先保存再生成”。
- 上传素材、手工输入 URL、删除图片后，表单值与预览区实时联动。

改动位置：
- `src/pages/VideoRemixTaskDetailPage.tsx`
- `src/features/video-remix/form.ts`
- `src/pages/VideoRemixTaskDetailPage.test.tsx`

实现方向：
- 抽出统一的 `saveCurrentFormValues` 调用链，供：
  - 保存按钮
  - 下一步按钮
  - 生成 Prompt 按钮
- “下一步”改成异步动作：
  - 先 `validateFields`
  - 再 `saveVideoRemixTaskForm`
  - 成功后切到下一步
- 素材区增加“输入值变化 -> 预览数组变化”的统一映射，避免 URL 文本和预览状态分裂。

### 2. 参考视频预览标准比例

目标：
- 参考视频不再用固定 `h-40` 裁切。
- 改成标准 `16:9` 容器，保证预览更稳定。

改动位置：
- `src/pages/VideoRemixTaskDetailPage.tsx`

实现方向：
- 用 `aspect-video` 或等价容器。
- `video` 采用 `w-full h-full object-contain` 或适合的预览策略。
- 对比区的参考视频也统一同样比例，避免列表页、详情页视觉不一致。

### 3. 商品图 / 人物图展示优化

目标：
- 上传后展示更像素材面板，而不是“文本框 + 下方图片列表”。
- 保留输入框联动能力，但输入区弱化为辅助，不抢主视觉。
- 删除按钮换成组件库更清晰的卡片操作。

改动位置：
- `src/pages/VideoRemixTaskDetailPage.tsx`
- 如有必要新增：
  - `src/features/video-remix/components/AssetPreviewGrid.tsx`

实现方向：
- 使用 `antd` 组件组合：
  - `Card`
  - `Image`
  - `Space`
  - `Tag`
  - `Button`
  - `Empty`
- 商品图、人物图统一成素材卡片网格。
- 每个卡片提供：
  - 缩略图
  - 文件序号或类型标签
  - 删除按钮
- URL 输入保留为只读或辅助编辑区，并与预览双向联动：
  - 输入换行 URL -> 自动生成图片卡片
  - 删除卡片 -> 自动回写文本值

### 4. 列表页“查看成品”优化

目标：
- 列表中的结果列不只是一个普通链接。
- 让“查看成品”更明确区分：
  - 有成品：突出查看
  - 无成品：显示处理中 / 无结果 / 失败原因

改动位置：
- `src/pages/VideoRemixTasksPage.tsx`
- 对应测试文件（如已有列表测试则同步补）

实现方向：
- 结果列使用更清晰的按钮化展示：
  - 有视频时显示主操作按钮：`查看成品`
  - 可附带一个次级入口：`进入详情`
- 若不进入双态详情页，本轮先让“查看成品”直接新开视频地址，视觉上做成按钮或带图标链接。
- 失败态继续显示错误文案，但样式强化。

### 5. 底部悬浮操作区

目标：
- “上一步 / 下一步 / 保存”在长页面滚动时持续可见。
- 不再被内容撑到底部。

改动位置：
- `src/pages/VideoRemixTaskDetailPage.tsx`

实现方向：
- 把当前 `StepActions` 改成页面底部悬浮条。
- 采用：
  - `sticky bottom-0`
  - 独立背景、边框、阴影
  - 内层限制宽度与页面主体对齐
- 这样用户滚到素材区底部或 Prompt 区时，仍然能直接保存和切步。

## 测试补充方向

- 详情页：
  - 点击“下一步”会先保存，再切换步骤
  - 输入图片 URL 时，预览区实时回显
  - 删除图片卡片后，文本字段同步移除 URL
  - 生成 Prompt 仍然保持先保存后生成
- 列表页：
  - 有 `videoUrl` 时渲染“查看成品”优化后的入口
- 构建回归：
  - 保持 `npm run typecheck`
  - 保持 `npx vitest run -c vitest.video-remix-regression.config.ts`

## 建议执行顺序

1. 先改详情页底部操作区和“下一步先保存”链路。
2. 再改参考视频标准比例。
3. 再抽素材卡片展示区，统一商品图 / 人物图展示与删除交互。
4. 最后补列表页“查看成品”优化和测试。
