## Why

AI爆款工厂的账号体系是 SaaS 平台的基础底座，页面建设不能只覆盖注册登录，还必须支撑身份识别、企业协作、积分用量、协议签署、权限拦截与风控留痕。

当前仓库尚未建立 React 前端工程与账号体系页面契约，且后续会基于 Figma MCP 扫描设计稿落地页面，因此需要先补齐 OpenSpec 变更、PRD 约束与前端实现边界。

## What Changes

- 新增账号体系 React 页面建设的 OpenSpec 变更，用于约束后续页面、状态、API 契约与验收。
- 新增账号体系 PRD 补充文档，沉淀用户类型、认证状态、企业空间、角色权限、积分、协议、风控日志与后台审核约束。
- 明确 MVP 第一阶段页面范围：注册/登录、手机号绑定、个人中心、实名认证、企业认证申请、企业空间首页、成员管理、积分用量、协议签署、权限拦截。
- 明确 React 技术方向：使用 React + TypeScript 构建页面，优先保留 Figma MCP 设计稿扫描结果作为 UI 源输入，代码实现遵循轻量、可测试、可扩展原则。
- 不在本变更中实现真实短信、微信、实名认证、支付、文件上传和后台审核服务；这些能力先以 API 契约和 Mock 数据方式约束。

## Capabilities

### New Capabilities

- `account-system-pages`: 账号体系前端页面、状态流转、权限拦截与 API 契约。

### Modified Capabilities

- 无。当前 `openspec/specs/` 为空，本次为新增能力。

## Impact

- 前端：后续新增 React + TypeScript 页面、路由、状态管理、表单校验、权限守卫、Mock/API Client。
- 后端/API：需要提供账号登录、微信绑定、实名/企业认证、成员管理、积分流水、协议签署、操作日志等接口契约。
- 设计：Figma MCP 扫描结果将用于对齐视觉布局，但业务流程以本文档与原始产品设计文档为准。
- 合规：涉及实名、企业证照、肖像权、素材授权、协议快照与日志审计，页面必须避免仅靠前端控制权限。
