## ADDED Requirements

### Requirement: 追爆任务详情页必须提供步骤导航式工作流
系统 MUST 在追爆任务详情页中提供清晰的步骤导航，并将任务编辑流程组织为“素材上传和配置”“提示词”“视频生成”三个连续步骤。系统 MUST 支持用户在当前任务详情页内切换步骤，同时保持当前任务上下文不丢失。

#### Scenario: 打开任务详情时展示步骤导航
- **WHEN** 用户进入任意一个追爆任务详情页
- **THEN** 系统 MUST 展示当前任务的步骤导航
- **THEN** 系统 MUST 将步骤数量限制为素材上传和配置、提示词、视频生成三步
- **THEN** 系统 MUST 高亮当前所在步骤

#### Scenario: 用户从当前步骤进入下一步
- **WHEN** 用户在某一步点击“下一步”
- **THEN** 系统 MUST 进入该任务的下一个步骤
- **THEN** 系统 MUST 保持当前任务 ID、表单内容和任务状态上下文不丢失

### Requirement: 素材上传和配置步骤必须重组字段顺序与交互
系统 MUST 将素材上传和配置作为任务详情页的第一个步骤，并在该步骤内先展示素材上传相关字段，再展示内容方向相关字段。系统 MUST 删除视频摘要输入，并将商品图、人物图的交互统一为上传、预览和删除，不再向用户暴露直接输入 URL 的表单控件。

#### Scenario: 用户进入素材上传和配置步骤
- **WHEN** 用户进入追爆任务详情页的第一个步骤
- **THEN** 系统 MUST 先展示参考视频、商品图、人物图等素材输入区域
- **THEN** 系统 MUST 在素材输入区域之后再展示产品信息、口播文案和复刻方向等内容配置字段

#### Scenario: 页面展示商品图和人物图输入方式
- **WHEN** 用户查看商品图或人物图区域
- **THEN** 系统 MUST 提供上传入口
- **THEN** 系统 MUST 展示已上传素材的预览列表
- **THEN** 系统 MUST 提供删除单个素材的操作
- **THEN** 系统 MUST 不展示供用户手工输入素材 URL 的文本框

#### Scenario: 页面展示视频摘要字段
- **WHEN** 用户查看素材上传和配置步骤
- **THEN** 系统 MUST 不展示“视频摘要”输入字段

### Requirement: 复刻方向和内容字段必须提供更明确的引导
系统 MUST 在素材上传和配置步骤中增强内容输入引导。系统 MUST 为“复刻方向”增加备注说明，并为“产品信息”“口播文案”提供 AI 自动生成入口。

#### Scenario: 用户查看复刻方向字段
- **WHEN** 用户浏览“复刻方向”输入区域
- **THEN** 系统 MUST 在该字段附近展示补充说明
- **THEN** 说明内容 MUST 帮助用户理解应该描述目标效果，例如是否保留背景、替换产品或替换人物肖像

#### Scenario: 用户查看产品信息和口播文案字段
- **WHEN** 用户浏览“产品信息”或“口播文案”输入区域
- **THEN** 系统 MUST 为每个字段提供独立的“AI 自动生成”入口按钮

### Requirement: 提示词步骤必须展示并允许编辑当前提示词
系统 MUST 将提示词作为独立步骤展示，并允许用户查看当前任务已生成的提示词内容。若提示词已存在，系统 MUST 允许用户在页面内直接修改提示词后再继续生成视频。

#### Scenario: 任务已有生成提示词
- **WHEN** 用户进入提示词步骤且任务已有提示词结果
- **THEN** 系统 MUST 展示当前提示词内容
- **THEN** 系统 MUST 允许用户在当前步骤中编辑该提示词

#### Scenario: 任务尚未生成提示词
- **WHEN** 用户进入提示词步骤且任务尚无提示词结果
- **THEN** 系统 MUST 展示空态提示
- **THEN** 系统 MUST 提供生成提示词的动作入口

### Requirement: 视频生成步骤必须提供新老视频对比展示
系统 MUST 在视频生成步骤中同时突出参考视频和生成结果视频的关系。若任务已有结果视频，系统 MUST 提供新老视频对比展示，而不是只显示结果链接或单个播放器。

#### Scenario: 任务已生成结果视频
- **WHEN** 用户进入视频生成步骤且任务存在结果视频
- **THEN** 系统 MUST 展示参考视频区域
- **THEN** 系统 MUST 展示生成结果视频区域
- **THEN** 系统 MUST 以可感知对比关系的布局呈现这两个视频

### Requirement: 提示词生成和视频生成过程必须提供明确进度反馈
系统 MUST 为提示词生成和视频生成两个异步阶段提供明确的 loading 和进度反馈。系统 MUST 不只依赖按钮 loading，而要在步骤内容区域内展示任务状态、进度条或阶段说明。

#### Scenario: 用户触发提示词生成
- **WHEN** 用户在提示词步骤触发生成提示词
- **THEN** 系统 MUST 显示该阶段的处理中状态
- **THEN** 系统 MUST 在步骤内容区展示进度或阶段说明

#### Scenario: 用户触发视频生成
- **WHEN** 用户在视频生成步骤触发生成视频
- **THEN** 系统 MUST 显示该阶段的处理中状态
- **THEN** 系统 MUST 在步骤内容区展示进度条或等价的进度反馈
- **THEN** 系统 MUST 在任务失败时展示失败原因

### Requirement: 参考视频预览必须缩小为节省空间的展示方式
系统 MUST 缩小参考视频预览的默认展示占位，以节省素材步骤页面空间，同时保留可预览能力。

#### Scenario: 用户查看参考视频预览
- **WHEN** 用户在素材上传和配置步骤中查看参考视频
- **THEN** 系统 MUST 提供视频预览能力
- **THEN** 系统 MUST 使用比当前详情页更节省空间的预览布局
